Buttons

Examples of buttons.

Info

  • <button> should be used when doing some action that doesn't have a meaningful URL.
  • <a> should be used when you want the look of a button but are linking to a meaningful URL.
  • Button height and width will be dependent on the size of the content, but the content should always be centered both vertically and horizontally.
  • Buttons should have 40px of horizontal padding on each side of the content.
  • Extend the buttons to the full width of the container when on mobile and stack when there is more than one (TODO - Determine breakpoints)

No-no's

  • No orange buttons
  • No icons on buttons
  • Do not use <input type="submit">

Submission Spinner

  • inprogress

Use this spinner on any button that is being used to save or submit data.

<button class="sprk-c-Button sprk-c-Button--primary">
  <span class="sprk-c-Spinner sprk-c-Spinner--circle"></span>
</button>

Additional Info

  • Time out timing and time out errors are based on products or scenarios, but when a time out happens, an alert to the user with clear content and instruction is a required.
  • When the button is clicked the text is replaced by the spinner animation.
  • The button must stay the same size when the spinner is applied.
